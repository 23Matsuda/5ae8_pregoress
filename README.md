## 1. 卒業研究報告書：研究実施計画（暫定）
【実施計画】
本研究では、WebAuthnと秘密分散法（SSS）を統合した鍵管理システムの構築に向け、以下のスケジュールで研究を遂行する。なお、本計画はプロトタイプの実装状況および実験結果に応じて柔軟に見直すものとする。

フェーズ1：調査・設計（4月〜5月）
先行研究（Split-key FIDO等）の精読を行い、現在のパスキーが抱えるプラットフォーム依存性や攻撃経路の増加に関する課題を整理する。これに基づき、Keycloakをベースとした分散管理システムの詳細設計を行う。

フェーズ2：システム開発（6月〜9月）
KeycloakのSPI（Service Provider Interface）を利用し、独自の認証フローを実装する。8月から9月の夏季期間を重点開発期間とし、WebAuthn認証とSSSシェアの結合ロジックの統合テストを完了させ、動作可能なプロトタイプを構築する。

フェーズ3：評価実験・システム改善（10月〜11月）
構築したプロトタイプを用い、特定のアカウント凍結やデバイス紛失を想定した可用性評価試験、およびセキュリティ耐性テストを実施する。実験結果に基づき、UI/UXの改善およびロジックの修正を行う。

フェーズ4：成果のまとめ（12月〜1月）
得られた実験データを集計・分析し、本手法の有効性を検証する。最終的な卒業研究報告書を執筆し、口頭発表に向けた準備を行う。
